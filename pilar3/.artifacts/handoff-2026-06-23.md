## 8. What Would Prevent This Bug (from `/metrics` NameError)

The `PlainTextResponse` NameError was caused by referencing a name in a Python decorator parameter that was defined inside the decorated function body. The lesson: **never reference a name in a decorator parameter that's defined inside the decorated function body.** If a correct test seam existed for `create_health_app`, a test calling `GET /metrics` would have caught this immediately. The architecture currently tests `create_health_app` only via an integration test that mocks Redis — not via HTTP client against the actual app.
