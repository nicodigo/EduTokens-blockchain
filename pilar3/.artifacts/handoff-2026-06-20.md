## Pendiente para próxima sesión

### Crítico

1. **Bug NCT `StreamLostError`**: la conexión AMQP se corta durante el minado (idle timeout). El bloque se mina pero el hilo `block_loop` del NCT crashea. Solución: reconexión automática en `nct/nct.py` o aumentar `heartbeat` en RabbitMQ URL.

2. **RabbitMQ `no exchange 'blockchain'`**: al reiniciar RabbitMQ con PVC limpio, NCT debe recrear el exchange. Ya lo hace en el arranque, pero si NCT arranca antes que RabbitMQ, falla. Verificar orden o reintentos.

### Medio — CI/CD (Fase 5)

Crear `.github/workflows/ci.yml` (build + push de las 4 imágenes) y `.github/workflows/gitleaks.yml`. El Workload Identity para GitHub Actions ya está provisionado en `iam.tf` (SA `github-actions`). Falta crear el Workload Identity Pool en GCP.

- Video de demostración

## Decisiones de diseño vigentes

| # | Decisión |
|---|---|
| D1 | Mismo repo para infra y código |
| D2 | OpenTofu solo GCP, kubectl para K8s (separación limpia) |
| D3 | AMQPS con certbot wildcard (sin distribución de CA a workers) |
| D4 | LoadBalancer solo en RabbitMQ |
| D5 | NCT singleton (replicas: 1) |
| D8 | Sin NetworkPolicy (rompieron DNS y cert-manager) |
| D13 | initContainer en RabbitMQ para borrar `.erlang.cookie` (bug fsGroup) |
| D14 | Ingress en `apps`, servicios internos vía port-forward |
| D15 | Dominios separados: `edutokens.xyz` (HTTPS) vs `rabbitmq.edutokens.xyz` (AMQPS) |

## Bugs conocidos y sus fixes

| Bug | Fix | Archivo |
|---|---|---|
| RabbitMQ `Cookie file must be accessible by owner only` | `RABBITMQ_ERLANG_COOKIE` + initContainer que borra cookie | `rabbitmq-statefulset.yaml` |
| RabbitMQ `cacerts,undefined` con verify_peer | `cacertfile = /etc/ssl/certs/ca-certificates.crt` (Alpine) | `rabbitmq-configmap.yaml` |
| NCT `StreamLostError` al publicar bloque | Falta implementar. Reconexión en block_loop | `nct/nct.py` |
| RabbitMQ `no exchange 'blockchain'` | NCT recrea el exchange al arrancar | `nct/nct.py` |
| Let's Encrypt self-check timeout | Borrar Ingress principal durante emisión del certificado | Manual |

## Comandos útiles para la próxima sesión

```bash
# Acceder al cluster
gcloud container clusters get-credentials edutokens-cluster --zone us-central1-a --project edutokens-2026

# Ver estado
kubectl get pods -n infra
kubectl get pods -n blockchain
kubectl get pods -n g-compumundo

# Port-forward a NCT
kubectl -n blockchain port-forward svc/nct 8080:8080 &
curl localhost:8080/health
curl localhost:8080/status

# Port-forward a RabbitMQ Management
kubectl -n infra port-forward svc/rabbitmq 15672:15672 &

# Rebuild y push de imágenes
./pilar3/scripts/build-push.sh

# Redeploy completo
./pilar3/scripts/deploy-k8s.sh

# Reiniciar deployment en cluster GPU
kubectl -n g-compumundo rollout restart deployment worker-gpu

# OpenTofu (infraestructura)
cd pilar3/tofu && tofu plan
tofu output rabbitmq_static_ip
tofu output nginx_ingress_static_ip
```
